# GPS 三层滤波器设计文档

## 概述

本项目实现了一个三层级联 GPS 滤波器（`lib/gps_filter.dart`），用于在高频原始 GPS 数据流中剔除异常点并平滑轨迹抖动。滤波器通过 UI 开关控制，用户可在原始模式与滤波模式之间自由切换。

## 架构

```
GPS 原始数据流 (distanceFilter=0)
       │
       ▼
┌──────────────────┐
│ Layer 1: 精度阈值 │  accuracy > 25m → 丢弃
└──────────────────┘
       │ 通过
       ▼
┌──────────────────┐
│ Layer 2: 速度阈值 │  implied speed > 50m/s → 丢弃
└──────────────────┘
       │ 通过
       ▼
┌──────────────────┐
│ Layer 3: 卡尔曼滤波│  1D Kalman (lat/lng 独立)
└──────────────────┘
       │
       ▼
  滤波后坐标 → 写入轨迹 / 更新地图中心
```

## 滤波层级详解

### Layer 1: 精度阈值过滤

| 参数 | 值 | 说明 |
|------|-----|------|
| `accuracyThreshold` | 25.0m | GPS `accuracy` 超过此值的点直接丢弃 |

**原理：** GPS 芯片报告的 `accuracy`（68% 置信半径）反映当前定位质量。在室内、高楼遮蔽、隧道等环境下 accuracy 可能飙升到 50-100m，这些点对轨迹毫无意义，直接丢弃。

**阈值选择依据：** 户外开阔环境通常 3-10m，城市街道 10-20m，25m 能容忍轻度遮蔽同时剔除明显无效点。

### Layer 2: 速度阈值过滤

| 参数 | 值 | 说明 |
|------|-----|------|
| `maxSpeedMs` | 50.0 m/s | 约 180 km/h，超过视为跳点 |

**原理：** 计算相邻两点的隐含速度（distance / time_delta），若超出物理合理范围则判定为 GPS "飘飞"跳点。

**计算方式：**
```dart
speed = haversine_distance(lastPoint, currentPoint) / dt
```

**阈值选择依据：** 50m/s (180km/h) 覆盖高速公路驾车场景。步行/骑行场景中出现 50m/s 的隐含速度几乎 100% 是定位跳变。

### Layer 3: 一维卡尔曼滤波

对纬度和经度分别应用独立的一维卡尔曼滤波器。

| 参数 | 值 | 说明 |
|------|-----|------|
| `processNoise` (Q) | 2.0m | 每秒位置变化的不确定性 |

**状态模型：**
- 状态量：位置（纬度或经度）
- 测量量：GPS 原始坐标
- 测量噪声 R：`accuracy²`（GPS 报告精度的平方）

**算法步骤：**

1. **预测步（Predict）：**
   ```
   variance += Q² × dt
   ```
   dt 为距上次更新的时间间隔（秒），表示随时间推移，位置不确定性线性增长。

2. **更新步（Update）：**
   ```
   K = variance / (variance + R)
   state = state + K × (measurement - state)
   variance = variance × (1 - K)
   ```
   K 为卡尔曼增益：
   - 当 `accuracy` 好（R 小）时，K 接近 1，信任测量值
   - 当 `accuracy` 差（R 大）时，K 接近 0，信任预测值（上一帧位置）

**processNoise 选择依据：** 2.0m 适合步行（~1.2m/s）和骑行（~5m/s）场景。值越大响应越快但平滑度降低，值越小越平滑但运动中会产生滞后。

## 开关机制

| 模式 | distanceFilter | 滤波 | 适用场景 |
|------|---------------|------|---------|
| OFF（默认） | 2m | 无 | 一般使用，省电 |
| ON | 0m | 三层滤波 | 需要高精度平滑轨迹 |

**切换行为：**
- 切换时重启 GPS 流（因 `distanceFilter` 改变需要新流）
- 切换时调用 `reset()` 清空滤波器内部状态
- 新录制开始时也调用 `reset()`

**UI 位置：** 右侧按钮组，紧邻录制按钮。开启时按钮高亮青色。

## 日志设计

滤波器开启时，日志同时保留原始 GPS 值和处理结果，便于户外测试对比分析：

```
[12:00:01.234] GPS: 39.904200, 116.407400 acc=4.2m spd=0.3m/s   ← 原始值（始终打印）
[12:00:01.235] Track: #5 pts, +2.1m, total=15.3m                 ← 滤波后写入轨迹
[12:00:02.100] GPS: 39.904180, 116.407380 acc=32.0m spd=0.1m/s   ← 原始值
[12:00:02.101] GPS filtered out (acc=32.0m)                       ← 被精度阈值丢弃
```

## 参数调优指南

| 场景 | accuracyThreshold | maxSpeedMs | processNoise |
|------|-------------------|------------|--------------|
| 步行/跑步 | 20-25m | 10-15 m/s | 1.5-2.0m |
| 骑行 | 25m | 30 m/s | 3.0-4.0m |
| 驾车 | 30m | 50 m/s | 5.0-8.0m |
| 室内测绘 | 15m | 5 m/s | 1.0m |

当前默认值（25m / 50m/s / 2.0m）是步行+骑行通用配置。
